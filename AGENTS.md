# Project rules

- Keep all public pages, admin copy, metadata, README and new commit messages in natural English.
- Keep Chinese OCR and Chinese-language analysis support, but do not show Chinese system UI.
- Never invent diagnoses, credentials, universities, experts, certifications or research claims.
- Private screenshots and chat text stay local where technically supported; never publish or train on them.
- Do not commit secrets, tokens, user uploads, database files, feedback exports or message exports.
- Keep questionnaires relationship-specific: exactly ten questions for partner/dating, family, workplace and friendship.
- Assess repeated behaviour, power, boundaries, impact and safety; do not infer fault from who appears calm.
- Use preview before production. Do not deploy, merge main, bind a domain or alter permissions without explicit approval.
- Verify desktop and narrow mobile layouts after UI work, including themes, reduced motion and touch controls.
- Run lint, tests and build before committing.
